# 実験協力者募集掲示板システム 要件定義書（改定版）

## 1. はじめに
本ドキュメントは、豊橋技術科学大学の学生・研究室が被験者を募集するための「実験協力者募集掲示板（team_empiricism版）」の改定要件定義書である。
従来の仕様から、ユーザー認証の撤廃、UIの改善、募集期間の明確化などの変更を反映している。

## 2. システム概要
- **システム名**: 実験協力者募集掲示板
- **目的**: 学内における実験協力者の募集と応募を円滑にする。
- **主要な変更点（Ver 3.0以降）**:
    - ユーザーログイン・アカウント登録の廃止（全機能ゲスト利用可）。
    - 投稿・編集・削除機能の簡素化（パスワード認証）。
    - 募集期間（開始日・終了日）による管理への移行。
    - 視認性を重視したUIデザイン（タグ表示、項目順序の最適化）。

## 3. 機能要件

### 3.1. ユーザー管理
- **アカウント不要**: 閲覧者、投稿者ともにログインは不要。
- **投稿者認証**: 投稿時に設定する「4桁の数字（編集・削除用パスワード）」により、本人確認を行う。

### 3.2. 募集一覧・検索機能（掲示板トップ）
- **一覧表示**:
    - カード形式で実験を表示。
    - **カテゴリータグ**: カード最上部に表示（例：生体運動制御システム研究室）。
    - **タグ表示**: 「謝礼」「所要時間」「募集期間」「実施場所」を統一されたデザインのバッジで表示。
        - 順序: 謝礼 → 所要時間 → 募集期間 → 実施場所
    - **概要非表示**: 一覧画面では実験概要（本文）は表示せず、メタデータのみでスッキリさせる。
    - **リンク**: カード全体をクリック可能エリアとする。
- **検索・フィルタリング**:
    - **キーワード検索**: タイトル、主催者名、本文などを対象。
    - **カテゴリーフィルタ**: 研究室単位での絞り込み。
    - **ソート機能**: 新しい順、古い順、実施日が近い順（＝募集開始日が早い順）。

### 3.3. 募集詳細機能
- **表示項目順序**:
    1.  実験タイトル
    2.  実験概要
    3.  謝礼
    4.  所要時間
    5.  募集期間（開始日 ～ 終了日）
    6.  実施場所
    7.  応募条件
    8.  募集人数
    9.  主催者
- **ヘッダー**: 「〇〇の詳細情報」などの文言は排除し、ステータスバッジのみを表示。
- **投稿者メニュー**:
    - 「編集」「削除」ボタンを配置。
    - 編集は黒色ボタン（アイコン付き）、削除は赤色ボタン（アイコン付き）で強調。
    - クリック時にパスワード確認画面へ遷移。
- **応募ボタン**: 外部フォーム（Google Forms等）への外部リンクを表示。

### 3.4. 新規投稿機能
- **アクセス**: 誰でも投稿可能。
- **入力項目**:
    1.  実験タイトル
    2.  実験概要（テンプレート挿入機能あり）
    3.  謝礼（デフォルト値：「時給1160円」）
    4.  所要時間
    5.  募集開始日（カレンダー入力）
    6.  募集終了日（カレンダー入力）
    7.  実施場所
    8.  応募条件
    9.  募集人数
    10. 主催者名
    11. 研究室（カテゴリー）
    12. 応募用URL（必須）
    13. 編集・削除用パスワード（必須、4桁数字）
    14. ステータス（募集中/募集終了）

### 3.5. 編集・削除機能
- **認証**: 投稿時に設定したパスワード（4桁）が一致した場合のみ実行可能。
- **編集**: 新規投稿と同じフォームを使用。
- **削除**: 確認画面を経てデータを物理削除。

## 4. データモデル（主な変更点）
- **Schedule（日時）廃止**: 単一の `schedule` フィールドを廃止。
- **Start Date / End Date 追加**: 募集期間を管理するため、`start_date`（開始日）と `end_date`（終了日）を採用。時刻情報は持たない。
- **Organizer Name**: ユーザー紐付けがないため、テキスト形式で主催者名を保持。

## 5. UI/UX ガイドライン
- **テーマカラー**: 黄色（Warning Color）を基調とし、黒文字で視認性を確保。
- **バッジ・タグ**: 重要な情報はバッジスタイル（枠線付き）で統一し、情報の種類を明確にする。
- **ボタン**:
    - アクションボタン（応募、編集、削除）は塗りつぶし等の強調デザイン。
    - 投稿者アクション（編集・削除）は誤操作防止のため、色分けとアイコンを併用。
